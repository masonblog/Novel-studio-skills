# Hermes Agent 委派模式（delegate_task）适配指南

Hermes Agent 支持通过 `delegate_task` 工具派生上下文隔离的子代理。在 Hermes 中运行本工作流时，**不要用换帽模式**，按本文以委派模式运行：主 Agent 扮演总编（贯穿全程、唯一对用户），其余五个角色全部经 `delegate_task` 派发。

## 模式映射

| 角色 | 运行方式 | toolsets | 并行性 |
|------|----------|----------|--------|
| 总编 | 主 Agent 本体 | 全部 | — |
| 架构师 | delegate_task | `["terminal", "file"]` | 单任务（world→synopsis→outline 有依赖，一次委派完成三份） |
| 设定师 | delegate_task | `["terminal", "file"]` | 单任务；卡量大时可按类别批量并行（人物卡完成后，任务/场景/道具三类可 batch） |
| 写手 | delegate_task | `["terminal", "file"]` | **每批一个任务，批内串行成文**；严禁多批并行 |
| 编辑 | delegate_task | `["terminal", "file"]` | 单任务，必须等写手任务返回后再派 |
| 设计师 | delegate_task | `["terminal", "file"]` | 可 batch 并行（封面/各立绘/各插画互相独立），与下一批写作互不阻塞 |

## 委派任务单模板（goal + context）

Hermes 子代理**完全看不到主对话历史**，`goal` 与 `context` 是它唯一的信息来源。任务单必须自包含，缺一项都会导致子代理瞎猜。

```
goal: |
  扮演网络小说创作流水线中的{角色名}，完成{具体任务，如：第 2 批（第 6–10 章）正文起草}。
context: |
  ## 必读文件（按序读取，先读前两个再开工）
  1. {repo}/skills/novel-{role}/SKILL.md   ← 你的工作方法，严格遵循
  2. {repo}/skills/novel-studio/SKILL.md   ← 流程宪法（含分批铁律）
  3. {workspace}/project.yaml              ← 项目状态（只读，不得修改）
  4. {本任务具体输入文件清单，逐个列绝对路径：大纲段、设定卡、前情提要、上批末章…}
  ## 任务边界
  - 只产出 {目标目录} 下的文件，使用 {模板路径} 模板
  - 禁止修改：{上游目录列表}
  - 发现上游问题：在产出中以 > [!ISSUE] 记录，不要自行修改
  ## 验收标准
  {该角色 SKILL.md 中对应门禁/自检清单的要点}
  ## 返回要求
  完成后返回摘要：产出文件清单、{角色特定数据，如字数/卡数}、ISSUE 列表。
```

要点：

- **文件路径用绝对路径**：子代理有独立终端会话，不要假设工作目录。
- **角色 SKILL.md 永远放必读第一位**：角色纪律不靠 context 复述，靠子代理自己读技能文件——这保证六个角色在任何工具下行为一致。
- **返回摘要只用于总编决策**，一切实质产物必须落盘工作区（本来就是铁律，在 Hermes 下更是硬约束：摘要之外的内容会随子代理上下文一起销毁）。

## 串并行纪律

- 批内**写手 → 编辑严格串行**：编辑的输入是写手落盘的初稿。
- 不同批次之间串行（分批铁律：批末汇报获用户确认前不开下一批）。
- 设计师任务可与正文批次并行（`art_mode: incremental` 时），用 batch 模式一次派发多张物料。
- 子代理被禁用 `clarify`/`send_message`，**不能向用户提问**——所有需要用户决策的事项由子代理写进 ISSUE 返回，总编在批末汇报中转达。这与本工作流"总编唯一对用户"的设计一致。

## 推荐配置（~/.hermes/config.yaml）

```yaml
delegation:
  max_iterations: 100        # 默认 50；写手单批 5 章建议调高
  max_concurrent_children: 3 # 默认值即可（仅设计师/设定师 batch 用到）
  max_spawn_depth: 1         # 保持扁平：只有总编可委派，与本工作流一致
  child_timeout_seconds: 1200 # 长章节写作适当放宽空闲超时
```

`max_iterations` 不足时的兜底：写手任务被截断不会丢稿（逐章落盘），总编检查 `04-drafts/batch-NN/` 缺哪几章，**再派一个新写手任务补齐缺章**（任务单注明"第 N 章已存在，从第 M 章继续"），不要重跑整批。

## 故障排查

| 现象 | 原因 | 处置 |
|------|------|------|
| 子代理产物格式跑偏 | 任务单没把角色 SKILL.md 列为必读 | 检查 goal/context 模板第 1 条 |
| 子代理改了上游文件 | 任务边界未写禁改清单 | 补"禁止修改"段；toolsets 不要给 web 之外的多余工具 |
| 返回摘要说完成但文件缺失 | 子代理超时/迭代耗尽 | 以工作区文件为准核验，派补齐任务 |
| 子代理向总编要更多信息 | context 不自包含 | 按模板补齐文件清单重派，勿在多轮对话里挤牙膏 |
